# COLMAP Learning

A Python pipeline for 3D reconstruction using COLMAP. Takes a video or set of images and produces sparse and dense point clouds.

```
Video → Frames → Feature Extraction → Matching → SfM (sparse) → PatchMatch (dense) → fused.ply
```

---

## Requirements

- Python 3.10+
- [COLMAP](https://github.com/colmap/colmap/releases/latest) — download `colmap-x64-windows-cuda.zip` (or `nocuda`) and extract to `../colmap-bin`
- [MeshLab](https://github.com/cnr-isti-vclab/meshlab/releases/latest) — for viewing `.ply` files, extract to `../meshlab`
- ffmpeg — for frame extraction from video

Install Python dependencies:
```powershell
pip install -r requirements.txt
```

---

## Quick Start

### 1. Prepare a scene

```powershell
# Extract frames from a video at 5 fps
ffmpeg -i myvideo.mp4 -vf fps=5 .\scenes\myscene\images\frame_%04d.jpg
```

### 2. Run sparse reconstruction

```powershell
# For video input (sequential matcher)
python main.py --workspace .\scenes\myscene --sparse-only --sequential

# For unordered photos (exhaustive matcher)
python main.py --workspace .\scenes\myscene --sparse-only
```

### 3. Run dense reconstruction

```powershell
python main.py --workspace .\scenes\myscene --dense-only
```

### 4. View results

```powershell
# Dense point cloud in MeshLab
.\view_ply.ps1 -workspace .\scenes\myscene -type dense

# Sparse point cloud in MeshLab
python main.py --workspace .\scenes\myscene --export-sparse-ply
.\view_ply.ps1 -workspace .\scenes\myscene -type sparse

# Sparse in COLMAP GUI
$env:QT_QPA_PLATFORM_PLUGIN_PATH="C:\Users\timeu\OneDrive\Documents\GitHub\colmap-bin\plugins\platforms"; & "C:\Users\timeu\OneDrive\Documents\GitHub\colmap-bin\bin\colmap.exe" gui
```

## Project Structure

```
ColmapLearning/
├── main.py               ← pipeline (feature extraction, matching, SfM, dense)
├── extract_frames.sh     ← extract frames from video via ffmpeg
├── view_ply.ps1          ← open sparse or dense .ply in MeshLab
├── requirements.txt
├── COMMANDS.md           ← full command reference
└── scenes/
    └── <scene>/
        ├── images/       ← input frames (add your images here)
        ├── database.db   ← auto-generated by COLMAP
        ├── sparse/       ← sparse reconstruction output
        └── dense/
            └── fused.ply ← final dense point cloud
```

---

## Pipeline Flags

| Flag | Description |
|---|---|
| `--workspace <path>` | Scene folder containing `images/`. Defaults to `workspace/` |
| `--sparse-only` | Run steps 1–3 only (feature extraction, matching, SfM) |
| `--dense-only` | Run steps 4–6 only (undistort, PatchMatch, fusion) |
| `--sequential` | Use sequential matcher — always use this for video input |
| `--export-sparse-ply` | Export sparse cloud to `sparse/sparse.ply` |

See [COMMANDS.md](COMMANDS.md) for the full reference.

---

## Tips

- **Video input** → always use `--sequential`. Exhaustive matching on video causes bundle adjustment instability.
- **Low frame count** → increase fps when extracting (5–10 fps for a 20s video gives 100–200 frames, which is ideal).
- **Dense reconstruction is slow** — ~11s per frame on an RTX 5070 Ti. 100 frames = ~20 minutes.
- **Reset a scene** to start from scratch:
```powershell
Remove-Item .\scenes\myscene\database.db, .\scenes\myscene\sparse, .\scenes\myscene\dense -Recurse -ErrorAction SilentlyContinue
```
